# クライアント動作確認ガイド（本番に反映せずローカルで確認する）

| 項目 | 内容 |
| --- | --- |
| 作成日 | 2026-07-07 |
| 目的 | クライアント（LP・WEBアプリ・管理画面）の改修を、**本番にデプロイせず**ローカルで確認する。バックエンド・データは本番をそのまま利用する |
| 前提 | Docker Desktop 起動済み |

---

## 使い方

```sh
cd Client
docker compose up        # sassのwatchも一緒に立ち上がる
```

| サイト | URL | 内容 |
| --- | --- | --- |
| LP | http://localhost:8080/ | `public_html/` を配信 |
| WEBアプリ | http://localhost:8081/ | `public_html_app/` を配信。**CSV・APIは本番に接続** |
| 管理画面 | http://localhost:8082/ | `public_html_admin/` を配信。**APIは本番に接続** |

- HTML/JS/画像はホストのフォルダをそのままマウントしているので、**編集 → ブラウザリロードで即反映**（nginxはキャッシュ無効化済み）
- SCSSは `scss/style.scss` を編集すると sass コンテナが自動で各サイトの `style.css` にコンパイルする
- 確認が終わって本番に反映するときは、mainブランチにマージするだけ（GitHub Actions `deploy-client.yml` が自動デプロイ。§デプロイフロー参照）

## 仕組み — same-originプロキシ方式

本番APIのCORS許可オリジンは `https://app.pol-is.jp` 等の3ドメインだけなので、`localhost` のページからAPIを直接fetchするとブラウザにブロックされる。そこで**ローカルのnginxが本番へのリバースプロキシを兼ね、ブラウザからはすべて同一オリジンに見せる**。

```
ブラウザ（localhost:8081）
  ├─ /              → ローカルの public_html_app/（改修中のファイル）
  ├─ /csv/*         → nginxが https://app.pol-is.jp/csv/* へ中継（本番の最新CSV・読み取りのみ）
  └─ /api/*         → nginxが https://api.pol-is.jp/* へ中継（本番API）
```

- 設定はローカル開発用nginxのみ（`Client/nginx_app/default.conf` / `nginx_admin/default.conf`）。本番のCloudFront/S3には一切影響しない
- JS側は `location.hostname === "localhost"` のときだけ `/api/` を向く（`public_html_app/javascript/common.js` の `API_URL`、`public_html_admin/javascript/admin.js` の `API_BASE`）。本番配信時の挙動は従来と同一
- `/admin/*` のIP許可リストは、プロキシ経由でもこのMacの回線IPで評価される（自宅・コワーキングのIPは許可済みなのでそのまま通る）
- ローカルの `public_html_app/csv/`（gitに入っている古いスナップショット）は `/csv/` プロキシが優先されるため使われない

## できること

- テーマ一覧・検索・ソート・詳細・分析画面を**本番の最新データ**で確認
- pol.is埋め込みの表示確認（embed.jsは本番から読み込まれる）
- AIテーマ生成のステップ確認（`/theme/generate_*` — 本番Lambdaが応答）
- 管理画面の一覧表示・確認（本番のアクセスキーが必要）

## ⚠️ 注意 — 「本番のバックエンド」に繋がっている

ローカルなのは**見た目だけ**で、APIの先は本番。以下は実データに影響する。

| 操作 | 影響 |
| --- | --- |
| テーマ作成の最終ステップ（`/theme/post_draft`） | **本番に APPROVED 状態で登録され、15分以内にbatch-createがpol.isへ実投稿する。** UI確認は投稿直前のステップまでにする。誤って投稿した場合は15分以内に管理画面から対処（間に合わなければ `/batch/delete` で削除） |
| AIテーマ生成（`/theme/generate_*`） | OpenAI APIの実費が発生する（1回数円程度）。連打しない |
| 管理画面の承認・編集 | **本番の下書きを直接操作する。** 表示確認以外は本番運用と同じ心構えで |
| LPのお問い合わせフォーム | localhostからは送信不可（contact.pol-is.jp のCORS許可外）。表示確認のみ |

## 代替: オフライン・データ固定で確認したい場合

プロキシではなくローカルのCSVスナップショットで確認したい場合は、`Client/nginx_app/default.conf` の `location /csv/` ブロックを一時的にコメントアウトし、必要なら最新データを取得する:

```sh
cd Client
aws s3 sync s3://app.pol-is.jp/csv/ ./public_html_app/csv/ --exclude ".DS_Store" --exclude "*/.DS_Store" --delete
```

## トラブルシュート

- **ポートが使えない**: 8080〜8082 を他プロジェクトが使用していないか確認（`lsof -i :8081`）
- **CSVやAPIがタイムアウト**: nginxはDockerの内蔵DNS（`resolver 127.0.0.11 valid=30s`）で上流を都度再解決するため、CloudFront/API GWのIPローテーションには自動追従する（2026-07-08に恒久対応済み）。それでも失敗する場合は `docker compose restart web-app web-admin`
- **APIが481**: アクセスキー誤り。管理画面は本番のBATCH_ACCESS_KEY（`Server/web/.env` 参照）
- **admin APIが4xx（キー正しいのに）**: 回線のIPが許可リスト外（例: モバイル回線）。`ADMIN_ALLOW_IPS` は Terraformの `admin_allow_ips` で管理
