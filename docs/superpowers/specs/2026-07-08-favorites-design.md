# お気に入り機能 設計書

| 項目 | 内容 |
| --- | --- |
| 作成日 | 2026-07-08 |
| 対象 | WEBアプリ（ホーム画面・テーマ詳細画面） |
| 目的 | テーマをブラウザ内にお気に入り登録し、一覧で絞り込めるようにする |
| 制約 | 実装はローカル確認可能な状態で止める（デプロイはユーザー指示後） |

## 決定事項

1. **保存先は localStorage**（当初Cookie案から変更。容量4KB制限と「全HTTPリクエストに添付される」問題を回避。ブラウザ内保存・端末非引き継ぎというユーザー向け性質は同じ）
2. **キーは `conversation_id`**（詳細ページがURLクエリから直接持つIDのため、ページ間の照合が簡単）
3. **フィルタUIはカテゴリタブ先頭に「全て」「⭐️お気に入り」を追加**。デフォルトは「全て」（選択状態で表示）。お気に入りタブはカテゴリと排他、検索とは併用可
4. **初回保存時に注意ダイアログを表示**（一度きり）: ブラウザ内保存であり、端末・ブラウザ変更やプライベートブラウズでは引き継がれない旨を明記
5. ⭐️ボタンの配置: 一覧カードは**左下**(フッター左端)、詳細ページは**下部ボタン群**（共有ボタンと同型のアイコンボタン）

## データ仕様（localStorage）

| キー | 値 |
| --- | --- |
| `favorite_conversations` | conversation_id の文字列配列（JSON）。例 `["9knucbecnr","2azcnjfbut"]` |
| `favorite_notice_shown` | `"1"` = 初回注意ダイアログ表示済み |

- localStorage が使えない環境（プライベートブラウズの一部等）では機能を黙って無効化（☆は表示するがトグルは no-op でもエラーを出さない。実装は try/catch）

## コンポーネント（`javascript/favorites.js` 新規・両ページ共通読み込み）

- `isFavorite(cid)` / `toggleFavorite(cid)`（戻り値=トグル後の状態）/ `getFavorites()`
- `bindFavoriteButtons(root)`: `[data-favorite-cid]` 要素にクリックトグルを結線（`stopPropagation` + `preventDefault` でカード遷移を抑止）。表示は `bi-star`（未登録）⇔ `bi-star-fill`（登録済）、`aria-pressed` 同期
- 初回トグル時のみ注意ダイアログを動的生成して表示（フルスクリーン半透明オーバーレイ+中央カード+OKボタン。両ページで同一実装）

## 一覧（ホーム）仕様

- タブ列: `[全て][⭐️お気に入り][社会・政治]…[その他]`
  - 「全て」= 既存の `currentCategory = 0` の可視化。**初期表示で選択状態**
  - 「お気に入り」= 専用値（`data-category="fav"`）。選択中は `getFavorites()` に含まれるカードのみ表示
  - 既存の排他動作を踏襲（タブはどれか1つが選択状態）。検索ワードとはAND併用
- カード: フッター左端に `<span role="button" aria-pressed>` の☆ボタン（カード全体が `<button>` のためネスト回避）。タップで即トグル、カードへは遷移しない
- お気に入りタブ選択中に0件: `.article-container` に「お気に入りはまだありません。カードの☆で追加できます」を表示
- お気に入りタブ表示中に☆を外した場合、そのカードは即座に一覧から消える（フィルタ再適用）

## 詳細ページ仕様

- `bottom-menu-button-group` の共有ボタンの隣に ☆/★ アイコンボタン（`button.favorite-button.button.secondary`、共有と同型）
- URLクエリ `conversation_id` を対象にトグル。ホームに戻ると一覧の☆に反映されている

## 表示・スタイル（`scss/pages/home/_favorites.scss` 新規 + 詳細用少量）

- カードの☆: 白アイコン（グラデーション上）。登録済は `bi-star-fill`（同色・塗り）
- タブ「全て」「お気に入り」: 既存 `.category-button` と同スタイル（お気に入りは `bi-star-fill` アイコン+文字）
- 注意ダイアログ: 半透明黒オーバーレイ（z-index はモーダル層）+ 白カード角丸16px + OKボタン（`.button.secondary`）

## エラー・エッジ

- localStorage 例外（容量・無効化）: try/catch で無視（トグル失敗時は見た目も変えない）
- 存在しない conversation_id がお気に入りに残っている（テーマ削除後）: 単に一致するカードが無いだけで害なし。掃除はしない（YAGNI）
- 詳細ページに conversation_id が無い（直アクセス等）: ☆ボタンを非活性/非表示

## 動作確認（ローカルのみ・デプロイしない）

1. カードの☆トグル → リロードで保持 → 「お気に入り」タブで絞り込み → 検索併用 → 0件表示
2. 詳細ページの★とホームの状態同期（双方向）
3. 初回トグルで注意ダイアログが**一度だけ**出る（ホーム/詳細どちらが先でも）
4. ☆タップでカード遷移が起きない／お気に入りタブ中に☆解除で即消える
5. ブラウザ実測（スクリーンショット・座標）でのセルフレビュー、390px/320px

## スコープ外

- サーバー同期・アカウント連携（localStorageのみ）
- お気に入り数の表示・並び順のカスタマイズ
