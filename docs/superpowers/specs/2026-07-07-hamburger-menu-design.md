# ホーム画面ハンバーガーメニュー 設計書

| 項目 | 内容 |
| --- | --- |
| 作成日 | 2026-07-07 |
| 対象 | WEBアプリ ホーム画面（`Client/public_html_app/`）のみ |
| 目的 | 今後の改修項目の「受け皿」となる常設メニューを左下に設置する |
| 決定方法 | ビジュアルコンパニオンでモック比較（展開スタイル3案・ボタン構成3案） |

## 決定事項

1. **展開スタイル**: モーフ展開 — 円形ボタンがそのまま角丸パネルへ変形する（扇形・縦ポップ案は不採用）
2. **既存ボタンとの関係**: 「？使い方」「＋テーマを作成」は現状維持。ハンバーガーは新設項目専用の受け皿（収納・統合案は不採用）
3. **非展開時のボタン高さは既存ピルボタンと同一**（ユーザー要件）
4. **メニュー内はサイズ差をつける**: お問い合わせ=通常サイズ、利用規約・プライバシーポリシー=小さく表示。LPトップへのリンクは置かない
5. **設置範囲はホームのみ**（テーマ詳細画面は対象外。将来の拡張は可能な設計にする）

## UI仕様

### 非展開時

- `#bottom-menu` の先頭（最左）に円形ボタンを配置。既存2ボタンはその右に現状どおり並ぶ
- 高さ: 既存 `.button` ピル（padding 14px）と同じレンダリング高。`aspect-ratio: 1 / 1` + `border-radius: 50%` で真円
- 配色: `--color-dark`（#202426）背景 + 白のハンバーガーアイコン（3本線、CSSで描画）。既存 `.button.secondary` と同トーン

### 展開時（モーフ）

- 円がそのまま左下を基点に **角丸24pxのパネル** へ変形
  - `width` / `height` / `border-radius` を `0.35s cubic-bezier(.4, 0, .2, 1)` でトランジション
  - パネルサイズ: 内容にフィットする固定値（目安 幅220px）。項目追加時に調整可
- ハンバーガーアイコンは × へモーフ（上下の線が45度回転・中央線はフェード）
- メニュー項目はパネル展開後に上から順にフェードイン（約60msずつの時差、下から8px スライド）
- 展開中、既存の「使い方」「テーマを作成」ボタンはフェードアウト（閉じると復帰）
- z-index: パネルは既存bottom-menuの上、モーダルより下

### メニュー内容（初期）

| 項目 | サイズ | 遷移先 |
| --- | --- | --- |
| ✉ お問い合わせ | 通常（既存ボタン相当） | `https://pol-is.jp/contact/`（同一タブ） |
| 利用規約 ・ プライバシーポリシー | 小さく横並び（フッターリンク風、白60%程度の色） | `/terms/`・`/policy/`（同一タブ） |

今後の改修項目（通知・マイページ等）は「お問い合わせ」の並び（通常サイズ）に追加していく。

### 閉じる操作

- ×ボタンタップ / パネル外タップ / Escキー
- `prefers-reduced-motion: reduce` 時はトランジションを無効化（即時開閉）

### アクセシビリティ

- トリガーは `<button aria-label="メニュー" aria-expanded="false|true">`
- パネルは `<nav>` 要素、項目は `<a>`

## 実装ファイル

| ファイル | 変更 |

| `Client/public_html_app/index.html` | `#bottom-menu` 内にトリガーボタン、直後にパネル（nav）を追加 |
| `Client/scss/pages/home/_hamburger.scss` | 新規。クラス名は汎用（`.hamburger-menu` 等）にし、将来他画面で再利用可能に |
| `Client/scss/pages/home/_index.scss` | 上記の読み込み追加 |

CSSは既存のデザイントークン（`--color-dark` 等のCSS変数、SCSSの共通mixin）を使い、ハードコードの色値を増やさない。

1. `cd Client && docker compose up` → http://localhost:8081/ （本番無反映のローカル確認。`docs/クライアント動作確認ガイド.md`）
2. 確認項目: 開閉アニメーション／既存ボタンとの高さ一致／既存ボタンのフェード復帰／リンク遷移／外側タップ・Escで閉じる／モバイル幅・PWAセーフエリアで崩れない／チュートリアルやモーダルとの重なり
3. 本番反映は main マージ（`deploy-client.yml` が自動デプロイ）
