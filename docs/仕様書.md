## 1. サービス概要

### 1.1 背景

Polis は「ブロードリスニング」（インターネット経由で意見を募り、テクノロジーを活用して合意形成を探る取り組み）を実現するSNSシステムである。

- 「賛成」「反対」の投票を通じ、レスバトルや誹謗中傷の応酬を生まない議論ができる
- 統計的分析による意見グループの可視化で「落とし所」となる意見を探ることができる

一方、オリジナルの Polis は日本のユーザーにとってユーザービリティが低く、そのままでは日本での普及が難しい。

### 1.2 目的

**「Polis」のシステムを日本に社会実装する。**

そのために、下記の課題を解決し、Polis をコアとして SNS に求められる周辺機能を実装する。

1. 日本語による UI・操作説明がなく、日本人ユーザーが初見で受け入れ難い
2. PC 前提の UI となっており、モバイルユーザーが快適に使用できない
3. 一覧・検索機能がなくテーマを見つけることができない
4. 投稿に手間が掛かり、思いついたテーマを議題にするまでのハードルが高い

### 1.3 実装方針

投票・意見グループ化などのコア機能は本家 Polis（pol.is）をそのまま利用し、本システムは以下を提供する。

- LP によるサービス周知
- 日本語 UI・チュートリアル
- テーマ一覧・検索機能
- モバイル最適化された体験（PWA 対応）
- AI アシストによる簡易なテーマ作成
- Polis の投票データを取得・集計して独自表示する分析画面

## 3. システム構成

### 3.1 ドメイン構成

| ドメイン | 役割 | ホスティング |
| --- | --- | --- |
| `pol-is.jp` | LP | S3（バケット `pol-is.jp`）+ CloudFront |
| `app.pol-is.jp` | WEB アプリ（PWA） | S3（バケット `app.pol-is.jp`）+ CloudFront |
| `api.pol-is.jp` | API（テーマ作成・バッチ・管理） | EC2（nginx + FastAPI、Let's Encrypt TLS） |
| `contact.pol-is.jp` | お問い合わせ受付 | API Gateway + Lambda + SES |
| `share.pol-is.jp` | 共有リンクリダイレクト | ―（共有 URL 生成に使用） |
| `pol.is` | 投票・意見グループ化のコア機能（本家 Polis） | 外部サービス |

AWS リージョン: `ap-northeast-3`（大阪）

### 3.2 アーキテクチャ概要

サーバーレスアーキテクチャ + バッチサーバー（EC2）の構成で、コストメリットと複雑な処理を両立する。

```
ユーザー
  │
  ├─ Route 53 → CloudFront → S3 ……… LP / WEBアプリ（静的サイト・CSVキャッシュ）
  │
  ├─ API Gateway → Lambda → SES …… お問い合わせ通知送信
  │
  ├─ pol.is（埋め込みAPI）………………… 投票・意見投稿・意見グループ化
  │
  └─ api.pol-is.jp（EC2 / VPC内）
       ├─ nginx（TLS終端・/dashboard/ 配信）
       ├─ FastAPI（テーマ作成AI・バッチ・管理API）
       ├─ MySQL 8.0（ユーザー・テーマ下書き）
       ├─ memcached
       └─ cron（5分間隔で投票情報集計・テーマ作成バッチを起動）
              │
              └─ pol.is からレポートCSVを取得 → S3へアップロード → CloudFrontキャッシュ無効化
```

- テーマ・投票データの実体は **S3 上の CSV ファイル**（クライアントが直接読む読み取り専用「データベース」）
- MySQL はユーザーアカウント（休眠機能）とテーマ下書き（`t_draft`）のみを保持する

## 4. クライアント仕様（`Client/`）

3 つの静的サイトで構成され、開発時はそれぞれ独立した nginx コンテナで配信される（`Client/docker-compose.yml`）。

| サイト | ディレクトリ | 開発ポート | 本番 |
| --- | --- | --- | --- |
| LP | `Client/public_html/` | 8080 | s3://pol-is.jp/ |
| WEB アプリ | `Client/public_html_app/` | 8081 | s3://app.pol-is.jp/ |
| 管理画面 | `Client/public_html_admin/` | 8082 | api.pol-is.jp/dashboard/（EC2 nginx 配信） |

### 4.1 LP（`public_html/`）

- `index.html` — タイトル「Polis JAPAN | 分断をなくすSNS」。セクション: `#hero` / `#problems` / `#features` / `#demo`（デモテーマカード → アプリへ誘導）/ `#info`。JSON-LD 構造化データを含む
- サブページ: `contact/`（お問い合わせ）、`terms/`（利用規約）、`policy/`（プライバシーポリシー）、`license/`（ライセンス）
- `javascript/contact.js` — お問い合わせフォーム。`https://contact.pol-is.jp/contact` へ `mail` / `name` / `content` を POST
- Google Search Console 検証ファイルを配置

### 4.2 WEB アプリ（`public_html_app/`）

PWA として登録可能（`manifest.json`、`javascript/service-worker.js`、スプラッシュ画像一式は `Asset/PWA/generate.zsh` で生成）。

#### 画面構成

| 画面 | ファイル | 概要 |
| --- | --- | --- |
| ホーム（テーマ一覧） | `index.html` + `javascript/home.js` | テーマカードの一覧表示・検索・絞り込み・ソート |
| テーマ詳細・分析 | `detail/index.html` + `javascript/detail.js` | Polis 埋め込みによる投票と、独自レンダリングの分析レポートのタブ切替 |
| テーマ作成モーダル | `javascript/create.js` | AI アシスト／手動の 2 モードによるステップ式作成ウィザード |

#### ホーム画面（`home.js`）

- `/csv/themes.csv` を取得しテーマカードを描画
- カテゴリ絞り込み（1: 社会・政治、2: お金・資産、3: 男女・性別、4: 外国人問題、5: テクノロジー、6: 医療・福祉、7: 生活、8: その他）
- キーワード検索（トークン分割・`data-searchtext` によるローカル検索）
- ソート: 新着順・古い順・人気順（投票数）

#### テーマ詳細画面（`detail.js`）

- URL クエリ `?conversation_id=` を `.polis` 要素の `data-conversation_id` に設定し、`https://pol.is/embed.js` を動的ロードして投票 UI を埋め込む
- 分析レポートは iframe ではなく独自実装:
  `/csv/report/report_<conversation_id>.csv` を取得し、コメントごとの賛成・反対・パスの円グラフと意見グループ別ツールチップを描画（賛成率順ソート、円グラフタップで意見内容・投票数を表示）
- 共有ボタン: `https://share.pol-is.jp/link?conversation_id=...` を生成しクリップボードへコピー
- 「詳細レポート」ボタン: 本家 `https://pol.is/report/<report_id>` へリンク

#### テーマ作成（`create.js`）

- **AIアシストモード**: テーマ入力 → 対立軸生成（`/theme/generate_axis`）→ シードコメント生成（`/theme/generate_comments`）→ 説明文生成（`/theme/generate_descriptions`）→ カテゴリ選択 → 投稿（`/theme/post_draft`）
- **手動モード**: テーマ・対立軸・コメント（8 件以上）・説明・カテゴリを手入力して投稿
- 下書きは localStorage に自動保存
- 投稿された下書きはサーバーの `t_draft` テーブルに登録され、バッチにより Polis へ反映される（§6 参照）

#### 共通ライブラリ（`common.js`）

- `API_URL = "https://api.pol-is.jp/"`、`USER_ACCESS_KEY`（bot 対策用アクセスキー）
- CSV パーサ（`loadCsvAsJson` / `parseCSV`）、Cookie・localStorage ヘルパー、`TutorialManager`（チュートリアル）、`ModalManager`、`CustomSelectManager`、クリップボード、ローディング表示

### 4.3 管理画面（`public_html_admin/`、配信コピーは `Server/admin/`）

- 運営者がアクセスキーを入力（Cookie 保存）し、`https://api.pol-is.jp/admin/info` からテーマ下書き一覧を取得
- 下書きの承認（`/admin/approve`）・編集（`/admin/edit`）を実施

### 4.4 スタイル（`Client/scss/`）

- 単一エントリ `style.scss` → 各サイトの `style.css` に sass コンテナ（node:20-alpine、`sass --watch`）でコンパイル
- 構成: `common/`（reset / color / font / components / layout / animation）、`pages/top/`（LP）、`pages/home/`、`pages/detail/`、`pages/admin/`
- パーシャルは `_` プレフィックスの命名規則

## 5. サーバー仕様（`Server/web/api/`）

### 5.1 技術スタック

| 項目 | 内容 |
| --- | --- |
| フレームワーク | FastAPI（async）+ uvicorn |
| 言語 / パッケージ管理 | Python 3.11 / Poetry |
| ORM | SQLAlchemy 2.0（async）+ aiomysql |
| DB / キャッシュ | MySQL 8.0 / memcached |
| AI 連携 | LangChain + langchain-openai（OpenAI GPT-5 系）、LangSmith（プロンプト管理・トレーシング）、LangGraph |
| スクレイピング | Selenium（Chromium）+ BeautifulSoup、httpx |
| Web 検索 | DuckDuckGo（ddgs） |
| AWS 連携 | aioboto3（S3・CloudFront） |

`main.py` では production 環境で `/docs` `/redoc` `/openapi.json` を無効化し、`TimeoutMiddleware`（300 秒）と CORS を設定。マウントされるルーターは `admin` / `batch` / `theme` の 3 つ（`user` ルーターは実装済みだが未マウントの休眠機能）。

リクエストは `core/common_route.py`（`CommonRoute`）で共通処理される: パスからルーター/API 名を解決 → 認証チェック → 非同期 DB セッション開始 → サービスの動的生成（`api.services.<router>`）→ S3 クライアント初期化 → ハンドラ実行 → 後処理。エラーは `trace_id` 付き JSON に正規化される。

### 5.2 API エンドポイント一覧

#### `/theme`（テーマ作成 API・`USER_ACCESS_KEY` で保護）

| エンドポイント | メソッド | 概要 |
| --- | --- | --- |
| `/theme/generate_axis` | POST | テーマから対立軸（立場）を AI 生成 |
| `/theme/generate_comments` | POST | 対立軸ごとのシードコメントを AI 生成 |
| `/theme/generate_descriptions` | POST | テーマ説明文を AI 生成 |
| `/theme/post_draft` | POST | ユーザー投稿の下書きを `t_draft` に登録（status: APPROVED） |

#### `/batch`（バッチ API・`BATCH_ACCESS_KEY` で保護）

| エンドポイント | メソッド | 概要 |
| --- | --- | --- |
| `/batch/update` | POST | 全テーマの最新レポート CSV を pol.is から取得し、投票数・コメント数に変化があれば S3 の `themes.csv` / レポート CSV を更新し CloudFront キャッシュ（`/csv/*`）を無効化 |
| `/batch/create_all` | POST | status = APPROVED の下書きを Selenium で pol.is 上に作成し、`themes.csv` に追加、status を POSTED に更新 |
| `/batch/delete` | POST | テーマを `themes.csv` から削除、レポート CSV を S3 から削除、下書きを論理削除 |
| `/batch/healthcheck` | GET | 死活監視 |

#### `/admin`（管理 API・`BATCH_ACCESS_KEY` + IP 許可リストで保護）

| エンドポイント | メソッド | 概要 |
| --- | --- | --- |
| `/admin/info` | POST | 下書き一覧取得 |
| `/admin/approve` | POST | 下書きを APPROVED に更新 |
| `/admin/edit` | POST | 下書き内容の編集 |

#### `/user`（未マウント・休眠機能）

`mail_check` / `create` / `login`（httponly の `session_id` Cookie を発行）/ `reload` / `edit` / `delete`。パスワードは `utils/security.py` でハッシュ化。

エラーはカスタムステータスコードを使用（例: 481 InvalidAccessKey、482 PolisReportUnavailable）。

### 5.3 AI テーマ生成（`services/theme.py`）

- モデル: `gpt-5`（テーマ・対立軸生成）、`gpt-5-nano`（reasoning_effort=low、コメント・説明・カテゴリ生成）
- プロンプトは **LangSmith から実行時に取得**（キー: `get_theme` / `get_axis` / `get_axis_standalone` / `get_comments` / `get_comments_standalone` / `get_description` / `get_description_standalone` / `get_category` / `get_background_selector`）
- 生成前に DuckDuckGo News で背景情報を収集しコンテキストとして付与
- LangChain LCEL（`RunnableParallel` / `RunnableBranch` / `.abatch`）でチェーン構成

### 5.4 テーマ自動作成（`services/batch.py`）

- Twitter / Togetter のスクレイピング（Selenium + BeautifulSoup）によるテーマ候補の自動抽出パイプラインを持つ
- `create_theme_on_polis()` は Selenium で pol.is の Web UI を操作し、サインイン → カンバセーション作成 → シードコメント投入 → 可視化有効化 → レポート作成を行い、URL から `conversation_id` / `report_id` を抽出する

### 5.5 環境設定（`api/configs/`）

環境変数 `APP_ENV`（`production` / `development` / `localhost`、デフォルト development）で環境フォルダを選択し、共通 `constants.py` にマージして読み込む。

| ファイル | 保持する設定 |
| --- | --- |
| `constants.py` | `API_BASE_URL` / `CLIENT_BASE_URL` / `ENCRYPT_SALT` / `BATCH_ACCESS_KEY` / `USER_ACCESS_KEY` / `ADMIN_ALLOW_IPS`（CIDR）/ `LOG_ENABLE_FLAGS` / `CORS_PARAMETERS` |
| `credentials.py` | `OPENAI_API_KEY` / LangChain（LangSmith）関連 / AWS アクセスキー / `CLOUDFRONT_DISTRIBUTION`（環境変数として注入） |
| `database.py` | DB ホスト・DB 名（`app_db`）・ユーザー・ポート・パスワード |
| `cache.py` | `MEMCACHED_HOST` |

環境フォルダは `.gitignore` で除外対象（§9.3 セキュリティ注意事項参照）。

## 6. データ仕様

### 6.1 CSV（S3 バケット `app.pol-is.jp` 配下、クライアントの実質的な読み取り専用 DB）

#### `csv/themes.csv` — テーマ一覧マスタ

| カラム | 説明 |
| --- | --- |
| `id` | テーマ ID |
| `category` | カテゴリ ID（1〜8、§4.2 参照） |
| `title` | テーマ名 |
| `description` | 説明文 |
| `conversation_id` | Polis のカンバセーション ID（埋め込み・レポート CSV のキー） |
| `report_id` | Polis のレポート ID（詳細レポートへのリンク用） |
| `votes` | 総投票数（バッチで自動更新） |
| `comments` | コメント数（バッチで自動更新） |
| `create_date` | 作成日 |

#### `csv/report/report_<conversation_id>.csv` — テーマ別集計レポート

Polis の `comment-groups.csv` エクスポート形式。

```
comment-id, comment, total-votes, total-agrees, total-disagrees, total-passes,
group-a-votes, group-a-agrees, group-a-disagrees, group-a-passes,
group-b-..., …
```

### 6.2 MySQL（`app_db`、`models/tables.py`）

| テーブル | 用途 | 主なカラム |
| --- | --- | --- |
| `t_draft` | テーマ下書き（承認・投稿待ち） | id, title, origin_url, origin_html, theme_name, theme_description, theme_comments, theme_category, conversation_id, report_id, post_status, status, 日時 |
| `t_user` | ユーザー公開情報（休眠） | id, name, profile, last_login_date, status, 日時 |
| `t_account` | 認証情報（休眠） | id, t_user_id, mail, password（ハッシュ）, session_id, last_api_date, status, 日時 |
| `t_user_add` | ユーザー付加情報（休眠） | id, t_user_id, user_prompt, status, 日時 |

- `post_status`（`models/types.py::PostStatus`）: `SUGGESTED=0` → `GENERATED=1` → `APPROVED=2` → `POSTED=3`、`REJECTED=101`
- 削除は status カラムによる論理削除

### 6.3 投票情報自動更新のデータフロー

1. cron コンテナが 5 分間隔で `POST /batch/update` を実行
2. S3 から `csv/themes.csv` をダウンロード
3. 各テーマの `https://pol.is/api/v3/reportExport/<report_id>/comment-groups.csv` を httpx で取得
4. 投票数・コメント数に変化があれば、レポート CSV と再生成した `themes.csv` を S3 にアップロード
5. CloudFront の `/csv/*` を無効化（`StorageS3.clear_cache`）
6. クライアントは CloudFront 経由で最新 CSV を直接取得

## 7. バッチ / cron（`Server/cron/`）

Alpine `crond` コンテナ（`Server/cron/Dockerfile`）から内部 API（`http://web:8000`）へ `BATCH_ACCESS_KEY` 付きで curl 実行（`crontab.txt`）。

| ジョブ | スケジュール | 内容 |
| --- | --- | --- |
| `POST /batch/update` | 5 分間隔（毎時 :03, :08, …） | 投票情報の取得・CSV 更新 |
| `POST /batch/create_all` | 5 分間隔（毎時 :00, :05, …） | 承認済み下書きの Polis への公開 |

バッチ処理量は管理テーマ数に比例し、ユーザーアクセスに依存して急増減しないため、オートスケール不要で安定性とコストメリットを両立している。

## 8. インフラ・デプロイ

### 8.1 本番構成

- **静的サイト**: S3 + CloudFront（LP: CF `E1L9GH70CTWAY6` / アプリ: CF `E1VDGUET2Z2OBX`）、Route 53 で DNS 管理
- **API**: EC2 上の Docker Compose（`web` + `nginx` + `db` + `cache` + `cron` + `certbot`）
- **AWS 認証**: localhost では環境変数のアクセスキー、本番 EC2 では IAM ロール
- **TLS**: Let's Encrypt（certbot）。更新手順は `Server/docs/SSL証明書の更新手順.md` 参照
- **nginx**（`Server/nginx/production.conf`）: 80→443 リダイレクト + ACME チャレンジ、WebSocket ヘッダ、`/dashboard/`（管理画面）は IP 制限（allow/deny）

### 8.2 開発環境

すべて Docker 上で動作（ルート `README.md` 参照）。

```sh
# サーバーサイド（Swagger: http://localhost:80/docs）
cd Server && docker-compose build && docker-compose up

# クライアントサイド（LP:8080 / アプリ:8081 / 管理:8082）
cd Client && docker-compose build && docker-compose up
```

- `Server/web/Dockerfile`: python:3.11-slim + Chromium / chromedriver + Noto CJK フォント（Selenium 用）。ビルド時に pip-licenses で `THIRD_PARTY_LICENSES.html` を生成
- テーブル設計・API 設計は Google SpreadSheet + Google Apps Script で管理し、ソースへ書き出す運用（概要書より）

### 8.3 デプロイ手順

CI は未導入。手動デプロイ（コマンド詳細は `Client/README.md`）。

- **クライアント**: `aws s3 sync` で `public_html/` → `s3://pol-is.jp/`、`public_html_app/` → `s3://app.pol-is.jp/`（**`csv/*` は除外**。CSV はバッチが管理）→ CloudFront 無効化
- **サーバー**: EC2 上で `docker compose -p PolisJAPAN -f docker-compose.yml -f docker-compose.prd.yml build / up`（`APP_ENV=production`）
- **SSL 証明書発行**: `docker-compose.ssl.yml`（nginx + certbot）で初回発行

## 9. セキュリティ・可用性

### 9.1 設計方針（概要書より）

- ユーザーアクセスがある部分（LP・アプリ・CSV 配信）はマネージドサービス（S3 / CloudFront）で可用性を確保。アクセス増加時も柔軟にスケール可能
- データを管理する内部システム（EC2 / VPC）はクローズド設計。SSH・MySQL 等の管理アクセスは開発者 IP に制限
- データキャッシュである CSV は S3 の自動バージョニングで保護
- Web サーバーへの改竄・ハッキングリスクは静的ホスティングにより低減
- AI によるテーマ下書き生成のみ負荷があるが、アクセス比率として極小の見込み（負荷増時は別途対応）

### 9.2 実装上の保護

| 対象 | 保護方式 |
| --- | --- |
| `/theme/*` | `USER_ACCESS_KEY`（クライアント埋め込みの bot 対策キー） |
| `/batch/*` | `BATCH_ACCESS_KEY` |
| `/admin/*` | `BATCH_ACCESS_KEY` + IP 許可リスト（CIDR、`utils/security.py`） |
| 管理画面 `/dashboard/` | nginx の IP 制限 |
| production の Swagger | 無効化（`/docs` 等を閉鎖） |

### 9.3 注意事項（要改善）

- 環境別 config フォルダ（`configs/localhost/` `configs/development/`）は `.gitignore` 対象だが、作業コピーには API キー・AWS キー等の実値が存在する。コミット履歴への混入有無の確認と、必要に応じたローテーションを推奨
- `services/batch.py::create_theme_on_polis` に pol.is のログイン情報がハードコードされている。環境変数等への外部化を推奨
- `USER_ACCESS_KEY` はクライアント JS に埋め込まれるため秘匿性はない（bot 対策目的の運用であることに留意）
