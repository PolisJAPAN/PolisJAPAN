プライベート版について

「社内・組織内のメンバーだけで使えるプライベートなPolisを作れませんか？」というお問い合わせを多くいただいています。

結論からお伝えすると、現在のPolisJAPANの仕組みでは、プライベート版（限定公開版）を提供することはできません。

このページでは、その理由と、次善の解決策として本家Polis（pol.is）を利用して「URLを知っている人だけが参加できる」議題を作成する方法をご紹介します。

1
現在の仕組みではプライベート版はできません

PolisJAPANは、投票や意見グループ化などのコア機能を本家Polis（pol.is）に任せ、その上に日本語UI・テーマ一覧・分析画面などを提供する構成になっています。

この構成のため、次の3つの理由からプライベート版を実現できません。

  • 1. ユーザー認証の仕組みがない

    PolisJAPANにはログイン機能がなく、すべてのページを誰でも閲覧できます。「特定のメンバーだけに見せる」ための入口を用意できません。

  • 2. テーマはすべて公開一覧に掲載される

    PolisJAPANで作成されたテーマは、公開のテーマ一覧に掲載されることを前提とした設計になっています。

  • 3. コア機能が外部サービス上にある

    投票や意見の分析は本家Polis（pol.is）のサーバー上で動いており、PolisJAPAN側でそのアクセスを制限することはできません。

2
「個別デプロイ」にはAGPLライセンスの壁があります

Polisはオープンソースソフトウェアなので、「自社のサーバーにPolisを構築（個別デプロイ）すればプライベート版が作れるのでは？」と思われるかもしれません。技術的にはその通りですが、ここにはAGPL-3.0ライセンスという大きなハードルがあります。

AGPL-3.0は「ネットワーク越しにサービスとして提供する場合でも、改変したソースコード全体を利用者に公開する義務が生じる」という、非常に強いコピーレフト型のライセンスです。

  • 商用利用・非公開運用には向きません

    商用利用そのものは禁止されていませんが、独自に開発したカスタマイズや社内システムとの連携部分も公開対象になり得るため、「自社だけのノウハウとして非公開にしたい」という一般的な企業利用とは相性がよくありません。

  • 構築・運用のコストも掛かります

    ライセンスの問題に加えて、サーバーの構築・保守・セキュリティ対応などの継続的なコストと専門知識が必要になります。

3
次善策：本家Polisで「URLを知っている人だけ」の議題を作る

厳密なアクセス制御はできませんが、本家Polis（pol.is）を直接使うことで、「URLを共有された人だけが参加できる」議題を作ることができます。

本家Polisで作成した議題（Conversation）は、どこかの公開一覧に自動掲載されることはありません。ユーザー認証こそできないものの、URLを知らない人はたどり着けないという意味で、「弱いプライベート」を確保した運用が可能です。

4
作成手順

本家Polisでの議題作成の流れは次の通りです。

  1. アカウントを作成する

    https://pol.is/home にアクセスし、画面右上の「Sign in」からアカウントを作成します。

  2. 議題（Conversation）を作成する

    ログイン後、「Create new conversation」ボタンをクリックします。

  3. Topic と Description を入力する

    Topic欄に議題のタイトルを、Description欄に議題の説明・背景を記入します。

  4. 準備が終わるまで議題を閉じておく

    「Conversation is open. …」というチェックボックスは、準備が完了するまで外しておきます。チェックを外している間、参加者は議題にアクセスできません。

  5. Seed Comments（初期意見）を登録する

    主催者側であらかじめ10〜20個程度の意見（論点）を登録しておくと、参加者が最初から投票しやすくなります。

  6. ユーザーインターフェースを設定する

    「Customize the user interface」のチェックボックス群で、運用方針に合わせて設定します。

    • ・「Participants can submit comments」：参加者による意見投稿を許可するか

    • ・「No comments shown without moderator approval」：投稿された意見を主催者の承認制にするか（限定メンバーでの運用では承認制をおすすめします）

    • ・Facebook / Twitterログインが不要な場合は、該当のチェックを外します

  7. 議題を公開状態にする

    準備ができたら「Conversation is open. …」にチェックを入れて、議題を有効化します。

  8. URLを参加者にだけ共有する

    左メニューの「Distribute」を開くと参加用URLが表示されます。このURLを、参加してほしい相手にだけメールや社内チャットなどのクローズドな手段で共有します。

  9. 運用する

    開催中は左メニューの「Moderate」（意見の承認・却下）、「Monitor」（参加状況の確認）、「Report」（結果レポートの表示）を使って運用します。

より詳しい画面付きの解説は、こちらの記事が参考になります。

5
この方法の注意点

この方法は、あくまで「URLを知らない人はたどり着けない」という弱い限定公開です。次の点にご注意ください。

  • 1. URLが漏れれば誰でも参加できます

    URLの転送や漏えいがあれば、意図しない人も参加できてしまいます。参加者ごとのログイン認証や本人確認はできません。

  • 2. URLはクローズドな手段で共有してください

    オープンなSNSや公開Webページに掲載すると、検索エンジンなどを通じて拡散する恐れがあります。メール・社内チャットなど、限定された手段で共有してください。

  • 3. データは本家Polisのサーバーに保存されます

    投稿された意見・投票データは、米国のThe Computational Democracy Projectが運営するpol.isのサーバーに保存されます。機密情報の取り扱いには十分ご注意ください。

  • 4. 厳密なアクセス制御が必要な場合には適しません

    「認証された社員のみ」「特定メンバーのみ」といった厳密なアクセス制御が必要な場合、この方法では実現できません。
